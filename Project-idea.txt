**PROJECT TITLE:** AI-Powered Credit Risk Scoring & Loan Approval System

---

## 🎯 OBJECTIVE

Build a full-stack fintech web application that evaluates loan applications using machine learning and financial rules.

The system must:

* Accept user financial data
* Predict probability of default using ML
* Calculate a credit score (300–900 scale)
* Categorize risk (Low / Medium / High)
* Apply rule-based checks
* Output decision: Approve / Reject
* Provide explainable reasons for decision

---

## 🧠 CORE CONCEPT

This system mimics real-world credit risk engines used by banks.

Decision = ML Prediction + Financial Ratios + Business Rules

## 🔄 SYSTEM WORKFLOW

1. User submits loan application
2. Backend validates input
3. Backend sends data to ML model
4. ML model returns default probability
5. Backend calculates:

   * Credit score
   * Financial ratios
6. Rule engine evaluates constraints
7. Final decision generated
8. Response sent to frontend
9. Data stored in database

---

## 📊 INPUT FEATURES

* income
* loan_amount

---

## 📐 FEATURE ENGINEERING

Calculate:

1. Debt-to-Income Ratio (DTI)
   = existing_debt / income

2. Loan-to-Income Ratio
   = loan_amount / income

3. Credit Utilization (approx)
   = existing_debt / (existing_debt + income)

4. Employment Stability Score
   = normalized employment_length

---

## 🤖 ML MODEL DETAILS

Train using:

* LendingClub or Kaggle dataset

Steps:

* Data cleaning
* Encoding categorical variables
* Feature scaling
* Train/test split
* Model training
* Save model (pickle)

Prediction:

* Input → model → probability_of_default

---

## 🔢 CREDIT SCORE LOGIC

Convert probability into score:

credit_score = 900 - (probability_of_default * 600)

Example:

* 0.1 → 840 (low risk)
* 0.5 → 600 (medium)
* 0.8 → 420 (high risk)

---

## 🚦 RISK CLASSIFICATION

* Low Risk → score > 750
* Medium Risk → 600–750
* High Risk → < 600

---

## 🛑 RULE ENGINE (CRITICAL)

Rules override ML if necessary.

Reject if:

* income < threshold (e.g., ₹20,000)
* DTI > 0.6
* loan_amount > 5x income
* too many existing loans (>5)

Approve only if:

* risk is Low or Medium AND rules satisfied
